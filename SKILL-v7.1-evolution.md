## 核心创新：智能缓存机制 🚀

### 缓存的价值：
- ⚡ **节省时间**：避免每次重复解析已知职位
- 💰 **降低成本**：减少Token消耗
- 📝 **保留配置**：用户确认的筛选要求持久保存
- 🔄 **增量更新**：只处理新增或变更的职位
- 📊 **历史追踪**：记录职位变化和招聘历史

## 启动流程（智能缓存检查）

### 🔍 步骤 0：技能启动时的缓存检查

每次启动技能时，首先执行缓存检查：

#### 情况 A：首次使用（无缓存）

提示用户：
```
❌ 未找到缓存文件

这是您首次使用该技能，我将：
1. ✅ 读取所有在招职位
2. ✅ 逐个解析职位描述
3. ✅ 与您确认筛选要求
4. ✅ 保存配置到本地缓存

预计耗时：10-15分钟（仅首次）

是否继续？
```

#### 情况 B：发现缓存（正常情况）

提示：
```
✅ 找到缓存文件

缓存信息：
📅 最后更新：[时间]
📊 已缓存职位：[数量]个
🎯 可直接处理职位：[数量]个

正在检查职位变化...
```

然后进行增量检查，识别：
- 🆕 新增职位（需要解析）
- 📝 职位信息更新（可选更新）
- ✅ 无变化职位（直接使用缓存）
- ❌ 已关闭职位（从缓存移除）

提供处理选项：
1. **快速模式**（推荐）：只处理新增职位，其他使用缓存（5分钟）
2. **完整模式**：重新解析所有职位（15分钟）
3. **仅更新模式**：只解析新增职位，不执行筛选
4. **查看并编辑缓存**：手动修改筛选要求
5. **直接处理**：使用缓存立即开始筛选（最快）

## 第一阶段：职位解析与确认

### 步骤 1.1：智能解析策略

根据缓存情况采用不同策略：

**策略 A：仅解析新增职位**（快速模式）
- 对新增职位进入详情页提取描述
- AI 智能解析要求
- 从缓存加载已知职位配置

**策略 B：从缓存加载已知职位**
- 跳过解析，直接使用历史配置
- 显示上次处理统计

### 步骤 1.2：进入职位管理页面
1. 打开 Boss直聘企业端
2. 点击左侧菜单的"职位管理"
3. 等待页面完全加载

### 步骤 1.3：读取所有开放中的职位列表
识别所有标记为"开放中"状态的职位，提取：
- 职位名称
- 工作地点
- 经验要求
- 学历要求
- 薪资范围
- 浏览数/沟通数/意向数

### 步骤 1.4：逐个点击职位获取详细描述

⭐**关键步骤**：对于每个需要解析的职位

1. **点击职位标题**进入职位详情页
2. **提取完整的职位描述信息**：
   - 岗位职责
   - 任职要求
   - 必备技能
   - 加分技能
   - 工作内容描述
   - 团队介绍

3. **智能解析职位描述**，提取结构化信息：

   **硬性要求（必须满足）**：
   - 学历：本科/硕士/博士
   - 专业：计算机/电子工程/数学等
   - 年限：X-Y年经验
   - 技术栈：具体的编程语言、框架、工具
   - 行业经验：特定行业背景
   
   **软性要求（优先考虑）**：
   - 公司背景：大厂/创业公司/特定公司
   - 项目经验：具体类型的项目
   - 领域专长：如 Agent、RAG、LLM 等
   
   **关键词提取**：
   - 技术关键词（PyTorch、TensorFlow、LangChain）
   - 业务关键词（推荐系统、NLP、CV）
   - 特殊要求关键词（开源贡献、论文发表）

4. **返回职位列表**，继续下一个职位

## 第二阶段：处理每个职位（循环遍历队列）

**⚠️ 重要**：第二阶段只在用户确认第一阶段的所有职位要求后才开始。

### 步骤 2.1：切换到目标职位的"推荐牛人"页面

1. 点击左侧菜单的"推荐牛人"
2. 点击右上角职位选择下拉框
3. 选择当前要处理的职位（按优先级顺序）
4. 等待推荐列表加载完成

### 步骤 2.2：批量收集并筛选候选人（v7.0 优化）⭐

**批量处理策略**（显著提升效率）：

**每批处理10-15个候选人**，分阶段完成：

#### 阶段1：批量收集基本信息（快速滚动）

向下滚动，收集10-15个候选人的可见信息：
- 姓名
- 学校背景
- 公司经历
- 学历层级
- **按钮状态**（"打招呼" / "继续沟通"）
- 在线状态

#### 阶段2：硬性筛选（快速淘汰）

对这一批候选人执行硬性筛选，按优先级淘汰：

**优先级1：已联系检查**（最高优先，100%准确）
```
IF 按钮显示"继续沟通"：
  → ⏭️ 跳过（已联系过）
  → 无需记录姓名，直接基于按钮状态判断
```

**优先级2：学校筛选**（硬性）
```
IF 学校不是985/QS前100：
  → ⏭️ 跳过（学历不符）
```

**优先级3：学历层级筛选**（硬性）
```
IF 职位要求硕士 AND 候选人只有本科：
  → ⏭️ 跳过（学历层级不符）
```

**优先级4：专业筛选**（硬性）
```
IF 专业明显不是计算机相关：
  → ⏭️ 跳过（专业不符）
```

**优先级5：公司背景筛选**（硬性或优先）
```
IF 职位有公司硬性要求 AND 候选人无目标公司经验：
  → ⏭️ 跳过（公司背景不符）
```

#### 阶段3：智能评分排序（筛选通过的候选人）

对通过硬性筛选的候选人进行**评分**（总分100分）：

**1. 学校层级分（30分）**
- QS Top 50：30分
- QS 51-100：25分
- C9高校（清北复交浙等）：28分
- 985院校：22分
- QS 101-200：18分

**2. 公司匹配分（40分）**
- 目标公司在职：40分
- 目标公司历史：35分
- 相关大厂：25分
- 其他优质公司：15分

**3. 活跃度分（15分）**
- 在线状态：15分
- 1小时内活跃：12分
- 今日活跃：10分
- 本周活跃：8分
- 其他：5分

**4. 求职意愿分（15分）**
- 离职-随时到岗：15分
- 在职-考虑机会：12分
- 在职-月内到岗：10分
- 在职-暂不考虑：5分

**按总分排序**，识别Top 3-5候选人

#### 阶段4：优先联系高分候选人

按评分从高到低，依次联系：
- 联系Top 3-5候选人（优先最优质的）
- 其余候选人记录备选

### 步骤 2.3：执行"打招呼"操作

对于选中的候选人：

1. 点击候选人卡片的"打招呼"按钮
2. 等待对话框弹出
3. 发送系统预设招呼消息
4. 确认发送成功（按钮变为"继续沟通"）
5. 关闭对话框，返回列表

### 步骤 2.4：持续遍历直到真正到底 ⭐⭐⭐

**正确的遍历终止条件**（v7.0关键优化）：

```
循环处理：
1. 重复步骤2.2-2.3（批量收集→筛选→评分→联系）
2. 向下滚动准备下一批候选人
3. 检查是否到底：

   ✅ 正确的终止信号（任一满足即停止）：
   - 页面显示"没有候选人了"
   - 页面显示"暂无更多"
   - 页面显示"已加载全部"
   - 连续3次滚动，页面高度不再变化

   ❌ 错误做法（不要使用）：
   - 看到重复候选人就停止（BOSS直聘不会循环显示）

4. 如果到底，结束当前标签页的处理
```

**关键提醒**：必须滚动到BOSS直聘平台显示明确的"没有更多"提示才能停止，否则会遗漏大量候选人！

### 步骤 2.5：切换到"新牛人"标签页

推荐牛人处理完后：
1. 点击页面顶部的"新牛人"标签
2. 这里是主动向该职位表达意向的候选人
3. 重复步骤 2.2-2.4 的批量筛选流程
4. 同样滚动直到显示"没有更多候选人"才停止
5. 完成"新牛人"标签页的全部候选人处理

**多轮策略建议**：
- 第一轮：彻底处理"推荐牛人"+"新牛人"两个标签页
- 第二轮（如需要）：刷新页面，重新遍历（按钮显示"继续沟通"的自动跳过）
- 平台会持续推送新候选人，定期检查即可

### 步骤 2.6：记录当前职位的处理结果

统计本职位的筛选数据：
```

  - [候选人1姓名] - [学校] - [公司]
  - [候选人2姓名] - [学校] - [公司]
```

### 步骤 2.7：更新缓存中的处理历史

```
💾 实时保存处理进度...

已更新缓存：
- 职位[X]新增联系记录：[N]人

```

### 步骤 2.8：移动到下一个职位

按照优先级顺序，重复步骤 2.1-2.7，直到所有职位处理完成。

## 特殊情况处理

- 等待3秒后重试

- 通知用户："检测到验证码，请手动完成验证后告诉我'继续'"

- 操作：记录进度，询问用户是否继续处理其他职位

- 如果是硬性要求信息缺失 → 跳过
- 如果是优先项信息缺失 → 继续评估其他条件

- 检测到登录页面
- 暂停自动化，通知用户："请登录后告诉我'已登录'继续"
